# 2026-04-30 — R4. Reference Gateway class and wrap_tool decorator

## Plan

- Add `src/agent_policy_gateway/gateway.py` with:
  - `Gateway` (mutable, simple) holding `policies`, `tool_specs`, `audit_writer`, `default_deny`.
  - `Gateway.execute(call, fn, *args, resource=..., **kwargs) -> (value, Decision)` —
    walks policies in order, returns the first-matching rule's effect, falls back to
    default-allow (or default-deny if configured), propagates taint via
    `taint.propagate`, writes the audit record, calls or raises.
  - `Gateway.wrap_tool(fn=None, *, tool_name=None, taint_spec=None, resource_arg=None)`
    — decorator that builds a `ToolCall` from the function arguments and routes
    through `execute`. Reserved kwargs (`apg_input_label`, `apg_agent_id`,
    `apg_call_id`, `apg_resource`) configure the call and are stripped before the
    wrapped tool sees them.
  - Exceptions: `GatewayError` base, `PolicyDenied`, `PolicyReview`. Each carries
    the `Decision` and `ToolCall` for audit-aware callers.
  - `AuditWriter` typing alias = `Callable[[ToolCall, Decision], None]`.
  - `Action.RATE_LIMIT` maps to `Verdict.ALLOW` in R4 (rate counters arrive in R5
    along with the on-disk audit log).
- Re-export new symbols from `agent_policy_gateway/__init__.py`.
- New test module `tests/test_gateway.py` covering: default-allow, default-deny,
  first-policy-wins ordering, DENY raises `PolicyDenied` (with attached Decision),
  REVIEW raises `PolicyReview`, RATE_LIMIT allows, audit writer is invoked once
  per decision (incl. denies), reserved kwargs are stripped, taint propagation
  through `ToolTaintSpec`, resource_arg matching, end-to-end web→summarize→email
  exfiltration is denied while internal flows are allowed, and bare/decorated
  invocation forms.
- Update `docs/design.md` with a short "Reference monitor (R4)" section that
  records the wrapper kwarg convention and the rate_limit-as-allow caveat.
- Move R4 from "Up next" to "Done" in `ROADMAP.md`; add a `### Added` bullet to
  `CHANGELOG.md`.
