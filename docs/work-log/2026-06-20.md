# Work log — 2026-06-20

## Roadmap item
**R39. `apg audit stats --fail-over PCT` CI gate.**

Add an opt-in threshold that turns the deny+review share into a CI signal: when the
combined flagged (deny+review) share of the filtered records *exceeds* `PCT` percent,
`stats` exits with a distinct non-zero code (**5**) after still printing the summary;
at or under the threshold it exits 0. Off by default (no `--fail-over` ⇒ exit codes
unchanged).

## Plan (smallest change)
- Add a pure, exported helper `audit_flagged_share(records) -> float` in `audit.py`
  returning the deny+review percentage (0.0 for an empty log), reusing the same
  count logic as `summarize_audit` / `audit_stats_dict`.
- Add `--fail-over PCT` (`type=float`, default `None`) to the `stats` subparser.
- In `_cmd_audit_stats`, after the text/JSON summary is printed, gate on the helper:
  return **5** when `--fail-over` is set and the share strictly exceeds `PCT`; else
  keep the existing return value. Empty log ⇒ share 0.0 ⇒ never over threshold.
- Gate runs after all existing filters (`--verdict/--since/--until/--tool/--agent`),
  so it composes with them.
- Tests in `tests/test_audit_stats.py`; ruff + mypy clean.

Add a `--csv` output mode (mutually exclusive with `--json`) emitting per-verdict
counts and percentages as CSV rows (`verdict,count,pct`) plus a trailing
`deny+review` row, for piping into a spreadsheet.

## Plan (smallest change)
- Add a pure, exported `audit_stats_csv(records, *, source=None)` helper in
  `audit.py` returning the CSV lines: a `verdict,count,pct` header, one row per
  `Verdict` in enum order (all four, even at zero hits), and a trailing
  `deny+review` row. Figures reuse `_pct`, matching the text/JSON renderers.
  Empty log → just the header (parallels the text/JSON empty-log shortcut).

  `_fail_over_code(...)` so `--fail-over` composes.
