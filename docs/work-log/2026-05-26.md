# Work log — 2026-05-26

## Roadmap item
**R20. Policy hot-reload / file-watch.** Allow a long-lived `Gateway` to reload its
policy when the YAML file changes on disk (opt-in, fail-closed if the new file is
invalid — keep serving the old policy and log the parse error).

## Plan (smallest change satisfying the acceptance criteria)
- Add a new module `src/agent_policy_gateway/reload.py` with a `WatchedPolicy`
  class plus a `watch_policy(path)` factory. `WatchedPolicy` is duck-typed to
  expose `first_match(call, *, resource=...)` exactly like `Policy`, so it drops
  straight into `Gateway.policies` with **no change to `gateway.py`** (the
  reference monitor only ever calls `policy.first_match`).
- On each `first_match`, cheaply check the file signature `(st_mtime_ns, st_size)`;
  if it changed, re-read and validate via the existing `load_policy`. On success
  swap in the new `Policy`. On failure (invalid YAML / schema), **keep the old
  policy**, remember the bad signature so we don't retry every call, and log the
  parse error through `logging.getLogger("agent_policy_gateway.reload")` plus an
  optional `on_error` callback.
- Export `WatchedPolicy` / `watch_policy` from the package `__init__`.
- Add `tests/test_reload.py` covering both acceptance criteria: a valid edit flips
  a rule's effect for subsequent calls without rebuilding the gateway; an invalid
  edit is rejected and the previous policy keeps serving (and the error is logged).
