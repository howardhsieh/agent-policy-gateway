- **R20. Policy hot-reload / file-watch.** — completed 2026-05-26 (commit `7ff9660`). A long-lived `Gateway` can now pick up edits to its policy file without restarting, opt-in and fail-closed. New `agent_policy_gateway.reload` module ships `WatchedPolicy` (plus a `watch_policy(path)` factory) — a thin view over a single policy file that is **duck-typed to `Policy.first_match`**, so it drops straight into `Gateway.policies` with **zero change to `gateway.py`** (the reference monitor only ever calls `policy.first_match`). On every `first_match` (and on the `policy` property) it stats the file and compares a `(st_mtime_ns, st_size)` signature against the last load; an unchanged file is a no-op, a changed file is re-read and validated through the existing `load_policy`. On a valid edit the new `Policy` is swapped in live and an `INFO` line is logged; on an invalid edit (bad YAML or schema) the **previously-loaded policy keeps serving**, the bad signature is remembered so a broken file is not re-parsed on every call (retried only once the file changes again, e.g. the operator fixes it), and the `PolicyError` is logged at `WARNING` through `logging.getLogger("agent_policy_gateway.reload")` and handed to an optional `on_error` callback. - **R7. OpenAI function-calling adapter** — completed 2026-05-03. Added `openai_adapter.py` exposing `OpenAITool` (frozen description: name, description, JSON Schema parameters, Python function, optional `resource_arg` / `taint_spec`), `openai_tool_specs(tools, *, prefix=None)` (builds the JSON `tools=[...]` array OpenAI Chat Completions / Responses expects), `wrap_openai_tools(gateway, tools, *, prefix=None)` (gateway-wraps every Python function and returns a name→callable map), and `dispatch_openai_tool_call` / `dispatch_openai_tool_calls` for converting model-produced `tool_calls` (dict-form *and* attribute-form, so both raw HTTP responses and the openai-python SDK shape work) into `role="tool"` messages. The real `openai` SDK is *not* a dependency — duck-typed against a tiny `id` / `function.name` / `function.arguments` protocol. Reserved kwargs (`apg_input_label`, `apg_agent_id`, `apg_call_id`, `apg_resource`) are orchestration-only and never appear in advertised JSON schemas; `apg_call_id` is auto-populated from `tool_call.id` so audit records line up with the model's view. Policy refusals (`PolicyDenied` / `PolicyReview`) become structured tool-role error payloads (`{"error":"policy_refusal", "verdict","rule_id","reason"}`) so the model sees feedback it can react to; customisable via `on_denied`. Tool exceptions and malformed input (`OpenAIToolCallError`) intentionally raise. 34 new tests in `tests/test_openai_adapter.py` cover construction validation, spec shape and prefixing, dispatch (allow / deny / review), JSON-parse errors, unknown-tool handling, taint propagation through the adapter, resource-arg binding (and `apg_resource` override), batched dispatch, and a worked web→email exfiltration scenario denied at send. 217/217 tests green; ruff clean. Public exports: `OpenAITool`, `OpenAIToolCallError`, `openai_tool_specs`, `wrap_openai_tools`, `dispatch_openai_tool_call`, `dispatch_openai_tool_calls`. Commit `4e56c5e`.
