# Work log — 2026-05-24

## Roadmap item

**R17. `redact` / declassify action.**

Add a policy action that *transforms* arguments (e.g. masks PII matched by the
taint condition) and lets the call proceed with a declassified label, recording
the transformation in the audit record.

Acceptance: a `strict-pii`-style rule redacts a matched field, the downstream
tool receives the masked value, and the audit record captures both the rule id
and the fact that redaction occurred.

## Plan (smallest change that satisfies the acceptance criteria)

- **policy.py** — add `Action.REDACT = "redact"`; add a frozen `RedactSpec`
  model (`fields`, optional `pattern`, `mask`, `declassify`, `add_label`) with a
  validator that rejects an empty `fields` list and an uncompilable `pattern`;
  add `Effect.redact: RedactSpec | None` and extend `Effect._check_shape` so
  `redact` is required iff `action == redact` and forbidden otherwise.
- **core.py** — add `Verdict.REDACT = "redact"`; add `Decision.redacted_fields:
  tuple[str, ...]` (serialized only when non-empty so existing audit records and
  round-trips are unaffected).
- **gateway.py** — map `Action.REDACT -> Verdict.REDACT`; in `_decide`, when the
  matched rule is a redact rule compute the declassified output label
  (`propagated - declassify + add_label`); in `execute`/`aexecute`, when the
  verdict is REDACT, mask the matched fields in both the downstream call
  arguments (via signature binding, with a kwargs-only fallback) and the
  audit-visible `call.args`, then stamp `decision.redacted_fields` before the
  audit writer runs. REDACT proceeds like ALLOW (no raise).
- **policies/redact-pii.yaml** — new shipped example with a redact rule.
- **tests** — extend `tests/test_policy.py` and `tests/test_gateway.py`.
