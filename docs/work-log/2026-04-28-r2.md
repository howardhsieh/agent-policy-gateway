# 2026-04-28 (R2) — Taint propagation algebra

## Roadmap item
**R2. Taint propagation algebra.**

Acceptance criteria from ROADMAP:
- `src/agent_policy_gateway/taint.py` implements label join (`∨`), subsumption (`⊑`),
  and propagation rules: outputs are tainted with the join of all input taints plus the
  tool's own source labels.
- Tests cover identity, commutativity, associativity, and a worked example
  (web_search → summarize → send_email refuses on high-web taint).

## Plan
- Add `src/agent_policy_gateway/taint.py` exposing:
  - `join(*labels)` — n-ary join over `TaintLabel`, with the empty-args case
    yielding the bottom label.
  - `subsumes(higher, lower)` — function form of the lattice order `⊑`.
  - `ToolTaintSpec` — dataclass declaring a tool's intrinsic source labels and an
    optional set of declassified sources it strips on output.
  - `propagate(input_labels, spec)` — pure propagation rule:
    `output = ((∨ inputs) ∨ spec.adds) \ spec.declassifies`.
  - `flows_to(label, allowed)` — convenience used by policies: True iff every
    source in `label` is also in `allowed`.
- Tests in `tests/test_taint.py` covering identity / commutativity / associativity
  for `join`, lattice order properties for `subsumes`, propagation behavior, and
  a worked end-to-end exfiltration example (`web_search` → `summarize` → `send_email`
  is denied because the input label carries `web` and the email sink does not allow it).
- Re-export the new helpers from the package `__init__`.
- Update `CHANGELOG.md` (Unreleased / Added) and move R2 in `ROADMAP.md` to Done.
- Add a short note in `docs/design.md` about declassification, since it's a new
  concept introduced by this module.

## Notes
- Chose to expose both method form (`label.join(other)`) and free-function form
  (`taint.join(*labels)`) because policy code reads better with the n-ary free
  function; gateway internals tend to use the iterable form (`join_all`).
- `ToolTaintSpec` is a new concept and required a short note in `docs/design.md`
  (declassification). The principle: a tool can drop labels its operator has
  audited it for, but the LLM cannot.
- Declassification operates on the *post-add* label set, so a tool that both
  adds and declassifies the same source is a no-op — handy for tests but
  probably a misconfiguration in real life.
