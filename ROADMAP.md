# Roadmap

The daily scheduled task reads this file, picks the **first unchecked item under "Up next"**,
implements it, runs tests, commits, and pushes. When an item is done, it is moved to "Done"
with the date and commit hash.

Keep items small enough to finish in a single run (roughly 1–3 hours of focused work).
Each item must include acceptance criteria so the daily task knows when it's done.

- **R9. Async gateway path** — completed 2026-05-05. New public exports: `Gateway.aexecute`, `Gateway.wrap_tool_async`, `wrap_mcp_session_async`. 40 new tests across `tests/test_gateway_async.py` (28 — allow / deny / review through `aexecute`, sync and async audit writers, audit-failure abort, resource binding, taint propagation, kwarg stripping, async exfiltration scenario denied at send) and `tests/test_mcp_adapter_async.py` (12 — discovery, prefixing, allow / deny, taint propagation, resource binding, audit integration, every error edge). 293/293 tests green; ruff clean. Commit `18bd6ab`.

- **R8. Anthropic tool-use adapter** — completed 2026-05-04. Added `anthropic_adapter.py` exposing `AnthropicTool` (frozen description: name, description, JSON Schema `input_schema`, Python function, optional `resource_arg` / `taint_spec`), `anthropic_tool_specs(tools, *, prefix=None)` (builds the flat Messages-API `tools=[...]` array — `{name, description, input_schema}`, no `type`/`function` wrapper), `wrap_anthropic_tools(gateway, tools, *, prefix=None)` (gateway-wraps every Python function and returns a name→callable map), and `dispatch_anthropic_tool_use` / `dispatch_anthropic_tool_uses` for converting model-produced ``tool_use`` content blocks (dict-form *and* attribute-form, so both raw HTTP responses and the anthropic-python SDK's `ToolUseBlock` shape work) into `tool_result` content blocks. The real `anthropic` SDK is *not* a dependency — duck-typed against a tiny `id` / `name` / `input` protocol. Reserved kwargs (`apg_input_label`, `apg_agent_id`, `apg_call_id`, `apg_resource`) are orchestration-only and never appear in advertised JSON schemas; `apg_call_id` is auto-populated from `tool_use.id` so audit records line up with the model's view. ``input`` arrives as a parsed JSON object but a JSON-encoded string is tolerated for robustness. Policy refusals (`PolicyDenied` / `PolicyReview`) become `tool_result` blocks with `is_error: true` and a structured `{"error":"policy_refusal","verdict","rule_id","reason"}` payload so the model sees feedback it can react to; customisable via `on_denied`. Tool exceptions and malformed input (`AnthropicToolUseError`) intentionally raise. 36 new tests in `tests/test_anthropic_adapter.py` cover construction validation, spec shape and prefixing, dispatch (allow / deny / review), JSON-string input tolerance, missing/wrong block-type errors, unknown-tool handling, taint propagation through the adapter, resource-arg binding (and `apg_resource` override), batched dispatch, and a worked web→email exfiltration scenario denied at send. 253/253 tests green; ruff clean. Public exports: `AnthropicTool`, `AnthropicToolUseError`, `anthropic_tool_specs`, `wrap_anthropic_tools`, `dispatch_anthropic_tool_use`, `dispatch_anthropic_tool_uses`. Commit `538b8d7`.
