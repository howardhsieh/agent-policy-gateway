## Notes
- Chose set-of-sources as the lattice element rather than a fixed enum; this lets
  applications grow their taxonomy (`web`, `crm.contact.email`, `pii.ssn`, …)
  without code changes.
- `to_json` is intentionally narrow (rejects non-core types with `TypeError`) so
  callers don't accidentally pass through raw dicts that bypass `to_dict`.
