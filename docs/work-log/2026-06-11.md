# Work log — 2026-06-11

## Roadmap item
**R31. `apg audit stats --verdict <v>` filter.**

Add a repeatable/single `--verdict {allow,deny,review,redact}` option to
`apg audit stats` that restricts the summary to records whose decision verdict
matches, computed *before* `audit_stats_dict` / `summarize_audit` run.

## Plan (smallest change)
- Add a pure, reusable helper `filter_by_verdict(records, verdicts)` in
  `audit.py` (exported in `__all__`) that returns only records whose verdict is
  in the requested set; `None`/empty returns all records unchanged.
- Wire a `--verdict` argparse option into the `audit stats` subparser using
  `action="append"` + `choices=[v.value for v in Verdict]` so it is repeatable,
  single-use, and rejects unknown verdicts with argparse's exit-2 choice error.
- In `_cmd_audit_stats`, apply the filter to the read records before passing
  them to `summarize_audit` / `audit_stats_dict`, so the count, span, top-rules
  and top-tools all reflect the subset; an empty-after-filter list summarizes as
  an empty log.
- Add tests covering: text filter to `deny`, `--json` filter, multi-verdict
  filter, empty-after-filter, the choices error (exit 2), and the helper itself.
