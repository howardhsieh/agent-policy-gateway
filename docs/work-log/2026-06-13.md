## Roadmap item: R34 ‚Äî `apg audit stats --since/--until` timestamp window

A natural continuation of the R29‚ÄìR33 `apg audit stats` line: a time-window
filter so an operator can scope the summary to an incident window without
pre-slicing the JSONL file. Mirrors the R31 `filter_by_verdict` design exactly.

### Plan (smallest change satisfying the acceptance criteria)
- Add a pure, reusable `filter_by_time(records, *, since=None, until=None)`
  helper to `audit.py` (no I/O), beside `filter_by_verdict`. Bounds compared
  lexicographically against `record.ts` (ISO-8601 UTC ‚Äî the same ordering
  `summarize_audit` already uses for the span min/max). Inclusive on both ends;
  a `None` bound is open on that side; `since=None, until=None` returns every
  record unchanged (back-compat).
- Export `filter_by_time` from `__init__.py` and import it in `cli.py`.
- Add `--since TS` / `--until TS` options to the `apg audit stats` subparser and
  apply the filter in `_cmd_audit_stats` after `filter_by_verdict` (filters
  compose; order is irrelevant). A window matching nothing summarizes as an
  empty log, exactly like an empty `--verdict` match.
- Add tests under `tests/test_audit_stats.py`.
