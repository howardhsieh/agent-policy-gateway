# 2026-05-03

## Roadmap item
**R7 — OpenAI function-calling adapter.**

The roadmap line carries no explicit acceptance criteria, so I'm interpreting
R7 by symmetry with R6 (MCP adapter): a one-call helper that mounts a set of
Python tools under a `Gateway` and emits the JSON tool specs that OpenAI's
Chat Completions / Responses API expects, plus a dispatcher that converts
model-produced `tool_calls` (with stringly-encoded JSON arguments) into
gateway-mediated invocations.

## Plan
- New module `src/agent_policy_gateway/openai_adapter.py`:
  - `OpenAITool` (frozen dataclass): `name`, `description`, `parameters`
    (JSON Schema), `function` (Python callable), optional `resource_arg`,
    optional `taint_spec`.
  - `wrap_openai_tools(gateway, tools, *, prefix=None) -> dict[name, callable]`:
    register taint specs, gateway-wrap each function via `Gateway.wrap_tool`,
    return the registered-name → callable map (advertised name → callable when
    no prefix; `<prefix>.<name>` when prefixed).
  - `openai_tool_specs(tools, *, prefix=None) -> list[dict]`: pure builder for
    the JSON sent to the API as `tools=[...]`. Output shape matches
    `{"type": "function", "function": {"name": ..., "description": ..., "parameters": ...}}`.
  - `dispatch_openai_tool_call(wrapped, tool_call, *, input_label, agent_id,
    resource, on_denied) -> dict`: parse one OpenAI-shape tool_call (dict or
    attribute form), invoke via `wrapped`, return a `role="tool"` message with
    JSON content. Policy denials become structured tool-role error messages;
    customisable via `on_denied`. Malformed input → `OpenAIToolCallError`.
  - `dispatch_openai_tool_calls(...)`: thin batched helper.
  - `OpenAIToolCallError` (ValueError subclass) for malformed model output.
- New tests under `tests/test_openai_adapter.py` covering: spec generation
  (with and without prefix), wrap registers gateway taint specs, allow path
  (round-trip through dispatch), deny path returns structured tool message and
  blocks the underlying call, review path, JSON-parse error, unknown tool,
  reserved-kwarg passthrough, resource_arg binding (and `apg_resource`
  override), attribute-form `tool_call` shape, taint propagation.
- Add `OpenAITool`, `OpenAIToolCallError`, `wrap_openai_tools`,
  `openai_tool_specs`, `dispatch_openai_tool_call`, `dispatch_openai_tool_calls`
  to the package public API and `__all__`.
- Append a "OpenAI adapter (R7)" section to `docs/design.md`.
- Update `CHANGELOG.md` and `ROADMAP.md`.
