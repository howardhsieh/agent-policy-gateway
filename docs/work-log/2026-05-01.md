# 2026-05-01 — R5: Append-only JSONL audit log + replay tool

## Roadmap item

**R5. Append-only JSONL audit log + replay tool.** Acceptance: `audit.py`
writes one JSON object per decision; CLI `apg-replay` reads a log and prints
a human-readable timeline.

## Plan

- Add `src/agent_policy_gateway/audit.py` with:
  - `JsonlAuditWriter`: a callable class compatible with the existing
    `AuditWriter = Callable[[ToolCall, Decision], None]` alias, opening the
    target file in append mode (`O_APPEND`) and flushing after each record so
    a crash leaves a recoverable log. Optional `fsync=True` for stronger
    durability. Context-manager support; explicit `close()`.
  - `AuditRecord` (frozen dataclass) with `ts` (ISO-8601 UTC),
    `call: ToolCall`, `decision: Decision`, plus `to_dict` / `from_dict`
    round-trip.
  - `read_audit(path)` generator that yields `AuditRecord` instances in
    file order, skipping blank lines and raising `AuditFormatError` on
    malformed JSON with the offending line number.
  - `replay_main(argv)` entry point implementing `apg-replay LOG`. Supports
    `--limit N` and `--verdict {allow,deny,review}` filters. Returns exit
    code 0 on success, 2 when the log file is missing.
- Wire `apg-replay = "agent_policy_gateway.audit:replay_main"` into
  `[project.scripts]` in `pyproject.toml`.
- Export the new public surface from `__init__.py`.
- Tests in `tests/test_audit.py` covering: writer appends JSONL, append
  semantics across opens, parent-directory creation, gateway integration
  (allow + deny paths), replay timeline output, `--verdict` and `--limit`
  filters, malformed-JSON error, missing-file exit code, context manager.
- Tighten the stale "R5 will add a counter" comment in `gateway.py` to a
  generic "future milestone" since rate-limit counting is not part of R5.
- Update `ROADMAP.md` (move R5 to Done) and `CHANGELOG.md`.
