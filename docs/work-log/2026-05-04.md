# 2026-05-04 — R8: Anthropic tool-use adapter

## Roadmap item
**R8. Anthropic tool-use adapter.** Mirror R7's surface for Anthropic's
Messages-API tool-use shape: build the `tools=[...]` array, gateway-wrap
Python tools, and translate model-produced `tool_use` content blocks into
gateway-mediated invocations + `tool_result` content blocks (with
`is_error` for policy refusals).

## Plan
- Add `src/agent_policy_gateway/anthropic_adapter.py` with:
  - `AnthropicTool` (frozen dataclass: `name`, `description`,
    `input_schema`, `function`, optional `resource_arg` / `taint_spec`).
  - `anthropic_tool_specs(tools, *, prefix=None)` returning the flat
    Messages-API shape `[{"name", "description", "input_schema"}]`.
  - `wrap_anthropic_tools(gateway, tools, *, prefix=None)` registering
    each tool with the gateway and returning a name → callable map.
  - `dispatch_anthropic_tool_use` / `dispatch_anthropic_tool_uses`
    accepting both dict-form and attribute-form tool_use blocks (so raw
    HTTP and the `anthropic` SDK's pydantic models both work),
    returning `tool_result` content blocks. `apg_call_id` is auto-set
    from the tool_use `id` so audit lines up with the model's view.
  - `AnthropicToolUseError` for malformed input or unknown tool names.
  - Reserved kwargs (`apg_input_label`, `apg_agent_id`, `apg_call_id`,
    `apg_resource`) honoured but never surfaced in advertised schemas.
  - Policy refusals (`PolicyDenied` / `PolicyReview`) become
    `tool_result` blocks with `is_error: true` and a structured
    `{"error":"policy_refusal", "verdict","rule_id","reason"}` payload;
    customisable via `on_denied`. Tool exceptions propagate.
- Add tests in `tests/test_anthropic_adapter.py` covering construction
  validation, spec shape + prefixing, dispatch (allow / deny / review),
  unknown-tool handling, dict-string/dict-mapping input shapes, taint
  propagation through the adapter, resource-arg binding (and the
  `apg_resource` override), batched dispatch, and the canonical
  web→email exfiltration scenario denied at send.
- Wire the public symbols into `src/agent_policy_gateway/__init__.py`.
- The real `anthropic` SDK is **not** added as a dependency — duck-typed
  against an `id` / `name` / `input` protocol on tool_use blocks.
