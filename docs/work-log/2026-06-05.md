# 2026-06-05 ‚Äî R25: `Selector` argument-value matching (`arg_equals`)

## Item

**R25. `Selector` argument-value matching (`arg_equals`).** Extend the policy
selector so a rule can match on a named argument's literal value
(e.g. `arg_equals: {channel: "#public"}`), not just tool name / identity /
taint / resource.

## Plan

- Add an optional `arg_equals: dict[str, str|int|bool] | None` field to
  `Selector` in `policy.py`, using strict scalar types so YAML floats /
  lists / mappings are rejected at load time, plus a validator rejecting
  empty/blank keys.
- Extend `Selector.matches` to check `call.args`: every listed key must be
  present and equal. Equality is type-strict between `bool` and `int`
  (`true` does not match `1`), mirroring YAML's distinct scalar types.
- Mirror the new clause in `cli.py`'s `_clause_rejection` so the
  `apg policy explain` trace honours it, and add a repeatable
  `--arg KEY=VALUE` option to `explain` (values parsed as YAML scalars so
  `--arg flag=true` is a bool, `--arg n=3` an int).
- Back-compat: absent `arg_equals` (`None`) leaves every existing code path
  byte-for-byte unchanged.
- Tests: match, mismatch, missing-arg, bool/int strictness, schema
  validation (bad value types, empty keys), YAML load, and the explain
  trace (match + each rejection shape + `--arg` parsing).

### Why R26 first
The policy DSL now has five selector dimensions (tool, identity, resource,
arg_equals, taint) and first-match evaluation, so it has become easy to write a
rule that can never fire ‚Äî either shadowed by an earlier, broader rule or
self-contradictory in its taint clause. `validate` only checks the schema;
nothing checked rule *reachability*. A conservative lint is the smallest
change that catches these real authoring bugs without false positives.

### Plan
- Add a `lint` subcommand to the `apg policy` argparse tree in `cli.py`.
- Component-wise conservative selector-generality helpers
  (`_pattern_at_least_as_general`, `_args_at_least_as_general`,
  `_taint_at_least_as_general`, `_selector_at_least_as_general`).
- `_taint_contradiction` for unsatisfiable taint clauses; `_lint(policy)`
  emitting W001 (shadowed) / W002 (never matches) findings in rule order;
  contradictory rules excluded from acting as shadowers.
- Exit codes 0 / 3 / 2 / 1 (clean / findings / missing file / malformed).
- Tests: `TestPatternGenerality`, `TestTaintGenerality`, `TestLint` in
  `tests/test_cli.py`, including a sweep asserting all shipped
  `policies/*.yaml` stay lint-clean.
- Propose R27‚ÄìR29 under "Up next" (tamper-evident audit log; `arg_equals`
  diff concretization; `apg audit stats`).
