### Why R22 first
The package is fully type-annotated throughout (`core`, `gateway`, every
adapter) and `mypy` is already a dev dependency, but the distribution ships **no
`py.typed` marker**. Under PEP 561 that means downstream projects running
`mypy` / `pyright` against the *installed* package get **no** type information —
the annotations are invisible. Adding the marker is the smallest possible change
that turns the existing annotations into a shipped, externally-usable type API.

  a guarded test builds the wheel and asserts `py.typed` is inside it (skipped
