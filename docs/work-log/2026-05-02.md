# 2026-05-02 — R6 MCP adapter

## Roadmap item
**R6. MCP adapter.** Given an MCP server, expose its tools through the gateway
with one line of code.

## Plan
- Add `src/agent_policy_gateway/mcp_adapter.py` with `wrap_mcp_session(gateway, session, *, taint_specs=None, resource_args=None, prefix=None)` that:
  - Duck-types the session against a sync `list_tools()` / `call_tool(name, arguments)` protocol — does not import the real `mcp` SDK (async path is reserved for R9).
  - Accepts both bare iterables and `.tools`-bearing wrappers from `list_tools()`, and both attribute- and dict-shaped descriptors.
  - Optionally namespaces tools under a prefix (`prefix="fs"` → `fs.read`).
  - Registers per-tool taint specs and resource-arg bindings on the gateway.
  - Returns a dict of registered tool name → gateway-mediated callable that strips the four reserved kwargs and forwards the rest as `session.call_tool(advertised_name, arguments=…)`.
- Export `wrap_mcp_session` from the package `__init__`.
- Add `tests/test_mcp_adapter.py` covering allow/deny paths, prefixing, taint propagation, resource-arg binding, the `apg_resource` override, descriptor shapes, error edges (missing methods, malformed names, duplicates), audit-writer integration, and underlying-session exception passthrough.
- Update `docs/design.md` with a short note on R6.
