# Work log — 2026-06-07

## Roadmap item
**R27. Tamper-evident audit log (hash chaining).** Each JSONL audit record
gains a `prev` field carrying the SHA-256 hex digest of the previous record's
serialized line; `apg-replay --verify` walks the chain and reports the first
broken link. Addresses adversary class **A8 (audit-log tampering)** in
`docs/threat-model.md`.

## Plan (smallest change that satisfies the acceptance criteria)
- Add an optional `prev: str | None` field to `AuditRecord`, serialized only
  when set, so legacy (unchained) records keep their exact prior on-disk shape
  and still round-trip.
- Add an opt-in `chain=True` flag to `JsonlAuditWriter`. When set, each written
  record carries `prev`; the genesis record uses a fixed 64-char sentinel
  (`GENESIS_PREV`). On re-open of an existing chained log the writer seeds the
  running digest from the last line so the chain continues unbroken.
- Add `verify_chain(path)` returning a structured result `(ok, lineno, reason)`
  and wire it into `apg-replay` behind a new `--verify` flag with a distinct
  exit code (4) for a broken chain (2 = missing file, as before).
- Decision recorded in design.md: chaining is **opt-in** (default off) to keep
  the legacy record shape byte-identical and all prior audit tests green.
