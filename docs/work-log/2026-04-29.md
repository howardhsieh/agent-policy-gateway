## Final report

Shipped **R3. Policy DSL v0 (YAML)**. Added an `agent_policy_gateway.policy`
module exposing frozen Pydantic models for a versioned YAML policy schema —
`Policy`, `Rule`, `Selector`, `Effect`, `TaintCondition`, and an `Action` enum
— plus `load_policy` / `load_policy_str` / `load_policies` which surface
structural and validation errors as a single `PolicyError` carrying the
source filename. Selectors use `fnmatch`-style globs for tool and resource
names, exact equality for agent identity, and three-clause boolean conditions
on the input taint label; effects validate effect-shape invariants
(`rate_limit` requires a positive `limit_per_minute`, other actions reject
one). Three example policies live under `policies/`: `default.yaml`,
`research-agent.yaml`, and `strict-pii.yaml`. Tests went from 65 to 109
(+44 in `tests/test_policy.py`). All green; ruff clean. Commit
`0569b45` — `https://github.com/howardhsieh/agent-policy-gateway/commit/0569b45aecb40a533de24f38e15dd520fecc04a4`.
