## Shipped
Added a top-N "busiest agents" breakdown to the audit summary. `summarize_audit`
now renders a `top agents (by hits, max N)` text block and `audit_stats_dict`
returns a `top_agents` `{name, count}` list, both built from a `Counter` of
`call.agent_id` and ranked by the existing `_top` helper (ties by name
ascending). Calls without an `agent_id` bucket under a new `_NO_AGENT`
`(unattributed - no agent_id)` label, mirroring `_NO_RULE`. Empty logs are
unaffected.

Design decision: `agent_id` is `str | None`, so unattributed traffic needed a
bucket. Chose to mirror the established `_NO_RULE` label pattern rather than
silently dropping such records, so the breakdown always sums to the record
count.
