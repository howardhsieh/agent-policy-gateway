## [Unreleased]

### Added
- **`apg policy diff` `arg_equals` scenario concretization (R28)**: rules gated only by an `arg_equals` clause are now exercised by `policy diff`. Previously the synthetic scenario matrix carried no call arguments, so a rule like `arg_equals: {channel: "#public"}` never matched its own scenario and two policies differing solely inside an `arg_equals` value reported `no decision changes`. The internal `_Scenario` now carries a hashable `args` slot (a key-sorted tuple of `(key, value)` pairs, keeping matrix dedup intact); each rule's `arg_equals` mapping is concretized into its scenario, the probe call is built with those args, and the per-change output line shows `args={...}`. `policy diff` also gains a repeatable `--arg KEY=VALUE` option (same `true`/`false` -> bool, integer -> int, else string coercion as `policy explain`); supplying `--arg` switches the diff into single-scenario mode. 6 new tests (`TestDiffArgEquals` in `tests/test_cli.py`). Test count delta: 593 -> 599 (+6); ruff clean.
- **Tamper-evident audit log — hash chaining (R27)**: `JsonlAuditWriter` gains an opt-in `chain=True` flag that adds a `prev` field to every JSONL record holding the SHA-256 hex digest of the previous record's serialized line, so truncation, deletion, and in-place edits become detectable — addressing adversary class **A8 (audit-log tampering)** in the threat model. The first record uses a fixed all-zero sentinel (`GENESIS_PREV`), and re-opening an existing chained log seeds the running digest from its last line so the chain continues unbroken across process restarts. `AuditRecord` grows an optional `prev` field serialized only when present, leaving legacy unchained records byte-for-byte unchanged and fully round-tripping. New `verify_chain(path)` walks the log and returns a frozen `ChainVerifyResult(ok, records, broken_line, reason)`; `apg-replay` gains a `--verify` flag that reports the first broken link with its line number and exits 0 (intact), 2 (missing file), or 4 (broken chain — a distinct code). Chaining is opt-in (default off) so the legacy record shape and every prior audit test are preserved. New public exports: `verify_chain`, `ChainVerifyResult`, `GENESIS_PREV`. 13 new tests (`tests/test_audit.py`: default-off no `prev` field, genesis + per-record digest, chained round-trip through `read_audit`, chain continuity across reopens, verify accepts an intact log, detects an in-place edit / deleted record / mid-line truncation, flags an unchained log, and the `--verify` CLI exit codes 0/4/2 plus unchained replay still working). Test count delta: 580 -> 593 (+13); ruff clean.
- **`apg policy lint <file>` CLI subcommand (R26)**: static dead-rule detection for a single policy file. Reports **W002** rules that can *never* match because their taint clause is self-contradictory (a source required by `all_of` but forbidden by `none_of`, or an `any_of` whose every source is forbidden by `none_of`), and **W001** rules *shadowed* by an earlier rule whose selector is at-least-as-general — under first-match evaluation the later rule is dead. The generality check is component-wise and deliberately conservative so every finding is a true positive: unset selector fields are most general, `tool`/`resource` globs are compared only as equal patterns or glob-vs-literal (`fnmatch.fnmatchcase`, never glob-vs-glob), `identity` is equality, `arg_equals` is constraint-subset with type-strict values, and taint conditions require `all_of`/`none_of` subset relations plus a guaranteed `any_of`. Contradictory rules cannot shadow. Exit codes: 0 clean, 3 findings reported, 2 missing file, 1 malformed policy. All four shipped `policies/*.yaml` are lint-clean, locked in by a directory-sweep test. 22 new tests (`TestPatternGenerality`, `TestTaintGenerality`, `TestLint` in `tests/test_cli.py`). Test count delta: 558 -> 580 (+22); ruff + mypy clean.
- **`Selector` argument-value matching — `arg_equals` (R25)**: a policy rule can now match on a named argument's literal value, not just tool name / identity / taint / resource. `Selector` gains an optional `arg_equals: {name: value}` mapping validated as `str -> (str | int | bool)` with strict scalar types (floats, lists, mappings, and nulls are rejected at load time; blank argument names are rejected). `Selector.matches` requires every listed argument to be present on the call and equal to the given value; comparison is type-strict between `bool` and `int` (YAML `true` does not match `1`), and an empty or absent `arg_equals` leaves matching byte-for-byte unchanged (full back-compat). The `apg policy explain` trace honours the new clause — a rejected rule reports either the missing argument or the mismatching value — and `explain` gains a repeatable `--arg KEY=VALUE` option to put arguments on the hypothetical call (`true`/`false` become bools, decimal integers become ints, everything else stays a string; the call line in the trace shows `args={...}` only when arguments were given). 20 new tests across `tests/test_policy.py` (`TestArgEquals`: match, mismatch, missing arg, int/bool values, bool/int strictness, multi-key conjunction, extra-args ignored, empty/absent no-ops, combination with other clauses, schema rejections, YAML load + first-match) and `tests/test_cli.py` (`TestExplainArgEquals`: trace match/mismatch/missing-arg lines, legacy call-line shape preserved without `--arg`, repeatable typed `--arg`, malformed `--arg` argparse exit, `_coerce_scalar` unit coverage). Test count delta: 538 -> 558 (+20); ruff + mypy clean.
- **`apg policy diff <old> <new>` CLI subcommand (R24)**: compares two policy files by the *decisions* they produce, not their text. A synthetic scenario matrix is derived from both policies' rule selectors (globs concretized, taint clauses satisfied, plus a neutral baseline) and every scenario is evaluated against both policies with `Policy.first_match`; scenarios whose `(rule id, action)` outcome changed are reported with their old and new decisions, or `no decision changes` when none did. Any of `--tool/--identity/--taint/--resource` narrows the diff to that single user-defined scenario, mirroring `explain`. Exit codes: 0 with or without changes, 2 on a missing file, 1 on a malformed policy.

Test count delta: 518 -> 520 (+2); ruff + mypy clean.

- **LangChain / LlamaIndex tool adapter (R21)**: new `agent_policy_gateway.langchain_adapter` module mirrors the OpenAI / Anthropic / MCP adapters for LangChain- and LlamaIndex-style agent frameworks, duck-typed so neither `langchain` nor `llama_index` is a runtime dependency (tested entirely against hand-rolled fakes). `wrap_langchain_tools(gateway, tools, *, taint_specs=None, resource_args=None, prefix=None)` takes an iterable of framework tool *objects* and returns a `{registered_name: callable}` map of gateway-mediated callables with semantics identical to `wrap_mcp_session` — the four reserved kwargs (`apg_input_label`, `apg_agent_id`, `apg_call_id`, `apg_resource`) are honoured and stripped before forwarding, per-tool `taint_specs` register under the (optionally `prefix`-namespaced) name, and `resource_args` binds an argument for `Selector.resource` matching with `apg_resource` taking precedence. The tool object is duck-typed two ways: the name comes from `tool.name` (LangChain `BaseTool`) or `tool.metadata.name` (LlamaIndex), and invocation tries, in priority order, `tool.invoke(args)` / `tool.run(args)` / `tool.call(**args)` / `tool.func(**args)` / `tool.fn(**args)` / `tool(**args)`, raising a clear `TypeError` if none is invocable; misshapen tools (no/empty/duplicate name) raise `ValueError` upfront. New public export: `wrap_langchain_tools`. 23 new tests (`tests/test_langchain_adapter.py`) cover all six invocation shapes, forwarding + reserved-kwarg stripping, allow/deny enforcement, resource-arg binding and override, taint-spec registration (bare + prefixed) and propagation, audit on allow + deny, the error edges, and the canonical web→email exfiltration scenario denied at send. Test count delta: 491 -> 514 (+23); ruff clean.
- **Policy hot-reload / file-watch (R20)**: a long-lived `Gateway` can now reload its policy when the YAML file changes on disk, opt-in and fail-closed. New `agent_policy_gateway.reload` module ships `WatchedPolicy` and a `watch_policy(path)` factory; `WatchedPolicy` is duck-typed to `Policy.first_match`, so it drops into `Gateway.policies` with no change to `gateway.py`. Each `first_match` compares a cheap `(st_mtime_ns, st_size)` file signature against the last load: unchanged file is a no-op, a changed file is re-read and validated via the existing `load_policy`. A valid edit swaps the policy in live (logged `INFO`); an invalid edit keeps the previous policy active (fail-closed), remembers the bad signature so it is not re-parsed every call, logs the `PolicyError` at `WARNING` through `logging.getLogger("agent_policy_gateway.reload")`, and invokes an optional `on_error` callback. The initial load in `watch_policy` is eager, so a broken starting file raises immediately. No background threads or OS file-watch hooks — the check is driven synchronously by gateway calls. New public exports: `WatchedPolicy`, `watch_policy`. 12 new tests (`tests/test_reload.py`). Test count delta: 479 -> 491 (+12); ruff clean.
- **Taint provenance chains (R19)**: a `Provenance` *side-channel* that records *where each taint source came from*, complementing `TaintLabel`'s *what*. `TaintLabel` is left untouched, so the lattice algebra and label value-equality stay byte-for-byte unchanged. Core adds two frozen value types — `ProvenanceEntry(source, tool_name, call_id)` and `Provenance(entries)` (an order-preserving, dedup-on-construction tuple with `origins(source)`, `add`, `merge`, `restrict_to(sources)`, and `to_dict`/`from_dict`) — plus an `input_provenance` field on `ToolCall` and an `output_provenance` field on `Decision`, both defaulting to the empty chain and serialized only when non-empty (legacy records and the allow/deny path round-trip unchanged). `taint.propagate_provenance(input_provs, spec, *, tool_name, call_id, output_label)` merges the inputs' chains, stamps a fresh entry for every source the tool `adds`, then restricts to the sources surviving in `output_label` (so declassification drops provenance too). `Gateway` gains an opt-in `track_provenance: bool = False` flag and a reserved `apg_input_provenance` kwarg; when enabled, `_match_rule` computes the output provenance and every `Decision` carries it (default off keeps existing `Decision`-equality tests untouched). `apg-replay`'s `format_record` prints an `origin:` line (`source<-tool@call_id`) when provenance is present. New public exports: `Provenance`, `ProvenanceEntry`, `propagate_provenance`, `PROVENANCE_KWARG`. 15 new tests (`tests/test_provenance.py`), including the acceptance scenario — a 3-hop `web_fetch -> summarize -> send_email` flow through a real `Gateway(track_provenance=True)` whose denied-send audit record names the originating `web_fetch` call id. Test count delta: 464 -> 479 (+15); ruff clean.
- **`apg policy` CLI — `validate` and `explain` (R18)**: new `apg` console script (`agent_policy_gateway.cli:main`, registered in `[project.scripts]` next to `apg-replay`/`apg-bench`) with a `policy` subcommand group. `apg policy validate <file>` parses and validates a policy YAML via the existing `load_policy`, printing a one-line OK summary and exiting `0` (valid), `2` (file missing), or `1` (malformed — with the line-located message `load_policy_str` builds from `yaml.YAMLError` / Pydantic `ValidationError`). `apg policy explain <file> --tool NAME [--identity ID] [--taint a,b] [--resource R]` builds a hypothetical `ToolCall` and prints a first-match trace through the policy's rules (`[MATCH ]`/`[ no  ]` with the rejecting clause/`[ skip ]`), then the matched rule id and effect — or a `no rule matched` line. The trace's per-clause checks mirror `Selector.matches` so the explanation always agrees with the gateway's real first-match. `main(argv)` returns the exit code for easy testing. New public export: `cli_main`. 16 new tests (`tests/test_cli.py`). Test count delta: 448 -> 464 (+16); ruff clean.
- **`redact` / declassify action (R17)**: a new policy effect that sits between `allow` and `deny` — instead of refusing a tainted call, it *transforms* the offending argument fields in place and lets the call proceed with a declassified label. Policy schema adds `Action.REDACT` and a frozen `RedactSpec` (`fields` — required, non-empty; optional `pattern` regex validated at load time; `mask` default `"[REDACTED]"`; `declassify` and `add_label` taint-source lists); a `pattern` masks only matching substrings (leaving surrounding text and non-string values intact) while the no-pattern form replaces the whole field. `Effect.redact` is required iff `action == redact` and forbidden otherwise. Core adds `Verdict.REDACT` and `Decision.redacted_fields` (serialized only when non-empty, so existing records and the allow/deny path are byte-for-byte unchanged). The gateway declassifies the output label (`propagated - declassify + add_label`) and, on the `execute`/`aexecute` paths, masks the matched fields both in the downstream call arguments (via signature binding, with a kwargs-only fallback) and in the audit-visible `call.args` before the audit writer runs — so the log never captures the raw value and records both the rule id and what was redacted. REDACT proceeds like ALLOW. New shipped example `policies/redact-pii.yaml`. New public export: `RedactSpec`. 21 new tests (`tests/test_policy.py`, `tests/test_gateway.py`). Test count delta: 427 -> 448 (+21); ruff clean.
- **Rate-limit enforcement runtime (R16)**: the policy `rate_limit` action is now enforced instead of mapping to ALLOW. New `agent_policy_gateway.ratelimit` module ships a `RateLimiter` — a sliding-window event log keyed by an arbitrary hashable key (the gateway uses `(agent_id, tool_name)`), with `peek(key, limit)` (read-only capacity check) and `try_acquire(key, limit)` (check-and-consume), an injectable `clock` (default `time.monotonic`) and configurable `window_seconds` (default 60). `Gateway` gains a `rate_limiter` field (default factory). `Gateway.decide` stays pure and *peeks* the limiter for a `rate_limit` rule (ALLOW while the window has room, DENY once full, no slot consumed); the stateful `execute`/`aexecute` paths route through `_decide(..., consume=True)`, which acquires a slot for an allowed call and turns an exhausted window into a `PolicyDenied` (rule id preserved, reason "rate limit exceeded: more than N calls/min for tool '...'") that is audited like any other refusal and never runs the wrapped tool. Only calls that actually run count against the budget; refused calls consume nothing. Budgets are per `(agent_id, tool_name)`. New public exports: `RateLimiter`, `DEFAULT_WINDOW_SECONDS`. 19 new tests (`tests/test_ratelimit.py` unit tests with an injectable fake clock for deterministic window expiry, plus a `TestRateLimitEnforcement` class in `tests/test_gateway.py` exercising the N-then-N+1 deny, audit of the refusal, the denied call never invoking the tool, window-expiry re-allow, per-agent-tool keys, the pure-`decide` peek, and async enforcement). Test count delta: 408 -> 427 (+19); ruff clean.
