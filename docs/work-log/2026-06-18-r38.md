# Work log — 2026-06-18

## Roadmap item
**R38. `apg audit stats` per-list top-N overrides.**

Add `--top-rules N`, `--top-tools N`, `--top-agents N` that override the shared
`--top` for just their list (each defaulting to `--top` when omitted), so an
operator can show, e.g., 3 rules but 10 agents.

## Plan (smallest change)
- `audit.py`: give `summarize_audit` and `audit_stats_dict` optional keyword
  caps `top_rules`/`top_tools`/`top_agents` (default `None`). Each resolves to
  `top_n` when `None`, so omitting all three is byte-for-byte unchanged. Use the
  resolved per-list cap in both the heading (`max N`) and the `_top(...)` slice.
- `cli.py`: add `--top-rules`/`--top-tools`/`--top-agents` (type=int,
  default=None) to the `stats` subparser; thread them into both renderer calls.
- `tests/test_audit_stats.py`: cover per-list overrides for the text renderer,
  the dict/`--json` renderer, the CLI, and the "omit ⇒ unchanged" invariant.
