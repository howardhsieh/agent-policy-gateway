# Work log — 2026-05-23

## Roadmap item
**R16. Rate-limit enforcement runtime.**

Map the policy `rate_limit` action to a real sliding-window counter keyed by
`(agent_id, tool_name)` so a rule with `limit_per_minute` actually denies the
N+1th call inside a 60s window, audited as a refusal. Counter must be
monotonic-clock based and unit-tested with an injectable clock so window
expiry is deterministic.

## Plan (smallest change satisfying acceptance)
- Add `src/agent_policy_gateway/ratelimit.py` with a `RateLimiter` class:
  sliding-window log keyed by an arbitrary hashable key, `peek` (read-only
  capacity check) and `try_acquire` (check-and-consume), injectable
  `clock` (default `time.monotonic`) and `window_seconds` (default 60).
- Wire it into `Gateway`: new `rate_limiter` field (default factory). Keep
  `decide` PURE — it *peeks* the limiter for `rate_limit` rules and reports
  ALLOW/DENY without consuming. The stateful entry points
  (`execute`/`aexecute`) *consume* a token via a private `_decide(..., consume=True)`;
  an exhausted window yields a DENY decision, audited, then `PolicyDenied`.
- Denied (over-limit) calls do NOT consume a token; only allowed calls do.
- Update the stale "rate limiting is deferred" docstrings in gateway.py.
- Export `RateLimiter` from the package `__init__`.
