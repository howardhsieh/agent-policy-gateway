# Work log — 2026-05-06

## Roadmap item: R10 — Worked end-to-end example: indirect prompt injection in a web-research agent

Acceptance criteria (from `ROADMAP.md`): `examples/indirect_injection/` shows a deliberately
injected web page, a naive agent that falls for it, and the same agent gated by
`agent-policy-gateway` refusing the exfiltration. Reproducible from a single command.

## Plan

- Add a new `examples/indirect_injection/` package containing:
  - `injected_page.py` (or a small HTML string constant) — a fake web page whose content carries
    an indirect prompt-injection payload telling the model to email internal data to an
    attacker-controlled address.
  - `tools.py` — three tiny in-memory tools the agent uses: `web_fetch`, `kb_lookup` (a fake
    internal knowledge base with a "secret" record), and `send_email` (writes to an in-memory
    outbox so tests can observe what would have been exfiltrated).
  - `agent.py` — a single `run_research_task(...)` function that simulates an LLM-driven
    research agent. To keep the demo deterministic and testable, the "LLM" is a simple stub
    that always follows the injected instruction it observes in fetched page content. This
    captures the behaviour we are trying to defend against without depending on a live model.
  - `naive.py` and `gated.py` — two thin wrappers that build the toolset (no gateway vs.
    gateway-mediated) and run the same `run_research_task`.
  - `__main__.py` — single-command entry point: `python -m examples.indirect_injection` runs
    both variants and prints "naive: exfiltrated <secret> to <attacker>" and "gated: refused
    by rule deny-web-to-email".
  - `README.md` — explains the scenario, how to run it, and what to look for.
- Wire the demo to use `policies/default.yaml` (which already contains `deny-web-to-email`) so
  it exercises the *shipped* policy rather than a one-off written for the example.
- Add a test module `tests/test_example_indirect_injection.py` that imports both runners and
  asserts:
  1. The naive agent successfully sends the injected email (exfiltration happens).
  2. The gated agent raises `PolicyDenied` and *no* email leaves the outbox.
  3. The audit log captures one DENY decision tagged with the `deny-web-to-email` rule id.
  4. The `__main__` entry point exits cleanly and produces the expected lines.
