# Changelog

All notable changes to this project will be documented in this file. The format is based
on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

## [Unreleased]

### Added
- **Async gateway path (R9)**: new `Gateway.aexecute(call, fn, *args, …)` and `Gateway.wrap_tool_async(…)` for `async def` tools. `aexecute` awaits the underlying callable's return when it is awaitable and awaits the audit writer's return when it is awaitable, so async log sinks plug in without a thread shim while synchronous writers keep working unchanged; the fail-closed-on-audit / DENY-or-REVIEW-raises contract matches `execute` exactly. `wrap_tool_async` mirrors `wrap_tool` (reserved kwargs, `resource_arg` binding, taint-spec registration) but returns an `async def` wrapper that routes through `aexecute`. New `agent_policy_gateway.mcp_async_adapter` module exposes `wrap_mcp_session_async(gateway, session, *, taint_specs=None, resource_args=None, prefix=None)` for duck-typed async MCP-style sessions exposing `await session.list_tools()` and `await session.call_tool(name, arguments)` (the real `mcp` SDK shape — the SDK remains an *optional* dependency). Sync descriptor helpers `_iter_tool_descriptors` / `_tool_name` are reused so prefixing, dict/attribute-shape support, and duplicate-name detection match `wrap_mcp_session`. Updated `mcp_adapter.py` docstring to point at the new async helper. 40 new tests across `tests/test_gateway_async.py` and `tests/test_mcp_adapter_async.py` cover allow / deny / review through `aexecute`, sync and async audit writers, audit-failure abort, kwarg stripping, resource binding (and `apg_resource` override), taint propagation, prefixing, dict and attribute descriptors, every misshapen-session error edge, and a worked async web→summarize→email exfiltration scenario that the gateway denies at send. Public exports: `Gateway.aexecute`, `Gateway.wrap_tool_async`, `wrap_mcp_session_async`.
- **Anthropic tool-use adapter (R8)**: new `agent_policy_gateway.anthropic_adapter` module exposing `AnthropicTool` (frozen tool description: name, description, JSON Schema `input_schema`, Python callable, optional `resource_arg` / `taint_spec`), `anthropic_tool_specs(tools, *, prefix=None) -> list[dict]` (builds the flat Messages-API `tools=[...]` array, `{name, description, input_schema}` — no `type`/`function` wrapper), `wrap_anthropic_tools(gateway, tools, *, prefix=None) -> dict[str, Callable]` (gateway-wraps each tool and registers any per-tool `ToolTaintSpec` under the prefixed name), and `dispatch_anthropic_tool_use` / `dispatch_anthropic_tool_uses` for converting model-produced `tool_use` content blocks into `tool_result` content blocks. Both dict-form payloads (raw HTTP responses) and attribute-form objects (the anthropic-python SDK's `ToolUseBlock` pydantic models) are accepted; the real `anthropic` package is **not** a dependency. ``input`` arrives as a parsed dict but a JSON-encoded string is also tolerated. Reserved kwargs (`apg_input_label`, `apg_agent_id`, `apg_call_id`, `apg_resource`) stay out of advertised JSON schemas; `apg_call_id` is auto-populated from the tool_use `id` so audit lines up with the model's view. `PolicyDenied` / `PolicyReview` become `tool_result` blocks with `is_error: true` and a structured `{"error":"policy_refusal","verdict":...,"rule_id":...,"reason":...}` payload (customisable via `on_denied`); tool exceptions and malformed input (`AnthropicToolUseError`) intentionally raise. 36 new tests in `tests/test_anthropic_adapter.py` cover construction validation, spec shape and prefixing, allow / deny / review dispatch (dict and attribute forms), JSON-string input tolerance, missing/wrong block-type errors, taint propagation through the adapter, resource-arg binding (and `apg_resource` override), batched dispatch, and a worked web→email exfiltration scenario. Public exports: `AnthropicTool`, `AnthropicToolUseError`, `anthropic_tool_specs`, `wrap_anthropic_tools`, `dispatch_anthropic_tool_use`, `dispatch_anthropic_tool_uses`.
- **OpenAI function-calling adapter (R7)**: new `agent_policy_gateway.openai_adapter` module exposing `OpenAITool` (frozen tool description: name, description, JSON Schema parameters, Python callable, optional `resource_arg` / `taint_spec`), `openai_tool_specs(tools, *, prefix=None) -> list[dict]` (builds the JSON `tools=[...]` array Chat Completions / Responses expect), `wrap_openai_tools(gateway, tools, *, prefix=None) -> dict[str, Callable]` (gateway-wraps each tool and registers any per-tool `ToolTaintSpec` under the prefixed name), and `dispatch_openai_tool_call` / `dispatch_openai_tool_calls` for converting model-produced `tool_calls` into `role="tool"` messages. Both dict-form payloads (raw HTTP responses) and attribute-form objects (the openai-python SDK's pydantic models) are accepted; the real `openai` package is **not** a dependency. Reserved kwargs (`apg_input_label`, `apg_agent_id`, `apg_call_id`, `apg_resource`) stay out of advertised JSON schemas; `apg_call_id` is auto-populated from the OpenAI `tool_call.id`. `PolicyDenied` / `PolicyReview` become structured tool-role error payloads (`{"error":"policy_refusal","verdict":...,"rule_id":...,"reason":...}`) so the model can react to refusals; tool exceptions and malformed input (`OpenAIToolCallError`) intentionally raise. 34 new tests in `tests/test_openai_adapter.py` cover construction validation, spec generation and prefixing, allow / deny / review dispatch, JSON parse errors, unknown tool, taint propagation through the adapter, resource-arg binding (and `apg_resource` override), batched dispatch, and a worked web→email exfiltration scenario. Public exports: `OpenAITool`, `OpenAIToolCallError`, `openai_tool_specs`, `wrap_openai_tools`, `dispatch_openai_tool_call`, `dispatch_openai_tool_calls`.
- **MCP adapter (R6)**: new `agent_policy_gateway.mcp_adapter` module exposing `wrap_mcp_session(gateway, session, *, taint_specs=None, resource_args=None, prefix=None)`. One call discovers every tool advertised by an MCP-compatible session (duck-typed against sync `list_tools()` / `call_tool(name, arguments)`; the real async `mcp` SDK is *not* a dependency, that path is reserved for R9), registers any per-tool `ToolTaintSpec`, and returns a `dict[str, Callable]` of gateway-mediated callables. Supports both bare-iterable and `.tools`-bearing `list_tools` shapes, attribute- and dict-form descriptors, an optional `prefix` for namespacing multiple sessions on one gateway (the advertised name is still used for the underlying call), per-tool `resource_args` for selector matching with the `apg_resource` override taking precedence, and the same four reserved kwargs as the core gateway (`apg_input_label`, `apg_agent_id`, `apg_call_id`, `apg_resource`) — stripped before forwarding. Misshapen sessions (missing methods, malformed descriptors, duplicate names) raise clear `TypeError` / `ValueError` upfront. 21 new tests in `tests/test_mcp_adapter.py` cover discovery, prefixing, allow + deny paths, taint propagation through registered specs, resource-arg binding (and override), audit-writer integration, and every error edge. Public export: `wrap_mcp_session`.
- **Append-only JSONL audit log + replay tool (R5)**: new `agent_policy_gateway.audit` module with `JsonlAuditWriter` (a callable compatible with the gateway's `AuditWriter` slot, opening the target file in append mode, flushing after every record, optional `fsync=True` for durability), `AuditRecord` (frozen `ts` / `call` / `decision` round-tripping through `to_dict` / `from_dict`), and `read_audit` (eager open, blank-line tolerant, raises `AuditFormatError` with line numbers on malformed JSON or missing keys). New `apg-replay` console script (entry point in `pyproject.toml`) reads a log and prints a human-readable timeline; supports `--verdict {allow,deny,review}` and `--limit N` filters and returns 0 / 2 / 3 for ok / missing-file / malformed-line. 26 new tests in `tests/test_audit.py` cover writer append semantics across reopens, parent-directory creation, single-line invariant under multi-line args, gateway integration on allow + deny paths, end-to-end exfiltration round-trip through the replay CLI, and the CLI's filter and exit-code behaviour. Public exports: `JsonlAuditWriter`, `AuditRecord`, `AuditFormatError`, `read_audit`, `replay_main`, `format_record`.
- **Reference monitor (R4)**: new `agent_policy_gateway.gateway` module with the `Gateway` class and `@gw.wrap_tool` decorator. The gateway evaluates policies in order (first match within a policy and across policies), propagates taint via `taint.propagate` to compute the output label on every decision, and writes an audit record before the wrapped tool runs (fail-closed-on-audit). Refusals raise `PolicyDenied` or `PolicyReview`, both carrying the originating `ToolCall` and `Decision`. The decorator recognises four reserved kwargs (`apg_input_label`, `apg_agent_id`, `apg_call_id`, `apg_resource`) and strips them before forwarding; `resource_arg` ties policy resource matching to a named parameter of the wrapped function via `inspect.signature`. `Action.RATE_LIMIT` maps to `Verdict.ALLOW` until the counter lands with R5. 27 new tests in `tests/test_gateway.py`, including a worked web→summarize→email exfiltration that the gateway denies at send. New public exports: `Gateway`, `GatewayError`, `PolicyDenied`, `PolicyReview`, `AuditWriter`, and the four reserved-kwarg constants.
- **Policy DSL v0 (R3)**: declarative YAML policies via the new
  `agent_policy_gateway.policy` module. Frozen Pydantic models —
  `Policy`, `Rule`, `Selector`, `Effect`, `TaintCondition`, `Action` —
  define a versioned schema with selectors over tool name (fnmatch
  glob), identity, resource glob, and three-clause taint conditions
  (`all_of` / `any_of` / `none_of`), and effects covering
  `allow` / `deny` / `review` / `rate_limit`. `load_policy`,
  `load_policy_str`, and `load_policies` parse and validate YAML,
  surfacing structural problems as `PolicyError`. Three example
  policies ship under `policies/`: `default.yaml` (baseline
  exfiltration prevention), `research-agent.yaml` (identity-gated
  publish, rate-limited search, reviewed external POST), and
  `strict-pii.yaml` (PII redaction-aware deny rules). 44 new tests in
  `tests/test_policy.py` cover schema validation, loader error paths,
  and first-match behaviour against the shipped examples.
- **Taint propagation algebra** (R2): `agent_policy_gateway.taint` module with
  `join` / `join_all`, `subsumes`, `flows_to`, `ToolTaintSpec`, and `propagate`.
  The propagation rule is `output = ((∨ inputs) ∨ spec.adds) \ spec.declassifies`,
  giving the gateway a pure function for computing per-call output taint. New
  helpers are re-exported from the package root. 31 new tests in
  `tests/test_taint.py` cover lattice algebra (identity, commutativity,
  associativity, idempotence), order properties of `subsumes`, propagation
  rules including declassification, and a worked end-to-end exfiltration
  example (`web_search → summarize → send_email` is denied).
- **Core data model** (R1): `TaintLabel`, `ToolCall`, `Decision`, `Verdict` in
  `agent_policy_gateway.core`, with `to_json` / `from_json` round-tripping and
  34 unit tests covering construction, value equality, lattice algebra
  (join idempotent / commutative / associative, subsumption), and JSON stability.
- Initial project scaffold: README, ROADMAP, LICENSE, design notes, daily runbook,
  smoke test, `pyproject.toml`.
