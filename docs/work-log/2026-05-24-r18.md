# Work log — 2026-05-24 (R18)

## Roadmap item

**R18. `apg policy` CLI: `validate` and `explain`.**

A standalone console script that (a) validates a policy YAML against the
Pydantic schema with friendly error messages and exit codes, and (b)
`explain` takes a tool name + identity + taint label and prints which rule
would match and why (first-match trace).

Acceptance: `apg policy validate policies/default.yaml` exits 0; a malformed
policy exits non-zero with a line-located message; `apg policy explain` prints
the matched rule id for a known web->email scenario.

## Plan (smallest change that satisfies the acceptance criteria)

- **src/agent_policy_gateway/cli.py** (new) — an `argparse` dispatcher mirroring
  the `apg-replay` / `apg-bench` style. Top-level `apg` with a `policy`
  subcommand group exposing `validate` and `explain`.
  - `validate <file>`: load via `load_policy`; exit `0` on success (print a
    one-line OK summary), `2` when the file is missing, `1` on a `PolicyError`
    (printing the friendly, line-located message that `load_policy_str` already
    builds from `yaml.YAMLError.problem_mark`).
  - `explain <file> --tool NAME [--identity ID] [--taint a,b] [--resource R]`:
    build a `ToolCall`, walk `policy.rules` in order, print a per-rule
    first-match trace (which clause matched/failed), and print the matched
    rule id + effect, or a "no rule matched" line. Exit `0`/`1`/`2` as above.
- **pyproject.toml** — register `apg = "agent_policy_gateway.cli:main"` under
  `[project.scripts]`.
- **src/agent_policy_gateway/__init__.py** — export `main` (as `cli_main`) for
  discoverability and testing parity with the other entry points.
- **tests/test_cli.py** (new) — cover: valid policy exits 0; missing file exits
  2; malformed YAML exits non-zero with a line-located message; `explain` prints
  the `deny-web-to-email` rule id for the canonical web->email scenario; explain
  with no match prints the default-applies line.
