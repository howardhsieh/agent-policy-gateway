# Work log — 2026-05-25

## Roadmap item

**R19. Taint provenance chains.** Extend taint tracking so each label can
remember which tool/source introduced it, enabling audit records and
`apg-replay` to answer "where did the `web` taint on this denied send
originate?". Acceptance: a 3-hop `web -> summarize -> email` flow produces an
audit record whose provenance lists the originating `web_fetch` call id.

## Plan (smallest change satisfying the acceptance criteria)

The hint allows extending `TaintLabel` *or* carrying a side-channel. A
side-channel is chosen so the lattice algebra (`join`, `subsumes`,
value-equality used across the existing taint tests) stays unchanged.

- **`core.py`** — add immutable `ProvenanceEntry(source, tool_name, call_id)`
  and `Provenance(entries)` value types with `to_dict`/`from_dict`,
  `origins(source)`, `merge`, and `restrict_to(sources)` helpers. Add an
  `input_provenance: Provenance` field to `ToolCall` and an
  `output_provenance: Provenance` field to `Decision`, both defaulting to
  empty and serialized only when non-empty (mirrors the `redacted_fields`
  pattern so legacy records keep their shape).
- **`taint.py`** — add `propagate_provenance(input_provs, spec, *, tool_name,
  call_id, output_label)`: merge input chains, stamp a new entry for every
  source the tool `adds`, then restrict to the sources surviving in the
  output label (so declassified sources drop their provenance too).
- **`gateway.py`** — add an opt-in `track_provenance: bool = False` flag and a
  reserved `apg_input_provenance` kwarg. When on, the gateway threads input
  provenance into the `ToolCall`, computes output provenance, and attaches it
  to the `Decision`. Default off so existing `Decision` equality tests are
  untouched.
- **`audit.py`** — `format_record` prints a `provenance:` line listing each
  surviving source and its originating `tool@call_id` when present.
- **`__init__.py`** — export `Provenance` and `ProvenanceEntry`.
- **`tests/test_provenance.py`** — Provenance algebra unit tests plus the
  acceptance scenario.
