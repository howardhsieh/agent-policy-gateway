## Roadmap item

**R28. `arg_equals` scenario concretization in `apg policy diff`.**

Closes the gap R25 deliberately left open: rules gated only by an `arg_equals`
clause never matched their own synthetic scenario in the `diff` matrix (the
scenarios carried no args), so two policies differing solely inside an
`arg_equals` value reported "no decision changes".

## Plan (smallest change satisfying the acceptance criteria)

- Grow `_Scenario` with a hashable `args` slot — a key-sorted tuple of
  `(key, value)` pairs (`_ScenarioArgs`) so dedup in `_build_matrix` keeps
  working.
- Concretize each rule's `arg_equals` mapping into its synthetic scenario in
  `_scenarios_from_policy`, so an arg-gated rule matches the scenario derived
  from its own selector.
- Thread the args through `_decide` (build the probe `ToolCall` with
  `args=dict(...)`), the neutral baseline, and the per-change output line.
- Add a repeatable `--arg KEY=VALUE` option to `policy diff`, reusing
  `explain`'s `_arg_pair` / `_coerce_scalar` so coercion (bool/int/str) is
  identical; `--arg` also switches `diff` into single-scenario mode.
